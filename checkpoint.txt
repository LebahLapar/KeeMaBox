### 🏗️ 1. Struktur Proyek Saat Ini
- **`/backend/`**
  - `app.py`: Server utama berbasis Flask (mengatur routing REST API, koneksi MongoDB, MQTT, dan Telegram).
  - `requirements.txt`: Dependensi Python (`flask`, `pymongo`, `paho-mqtt`, `requests`).
  - `Dockerfile`: Image builder untuk aplikasi backend.
  - **`/templates/`**:
    - `index.html`: UI Dashboard Owner (Tailwind CSS, dark mode, glassmorphism, custom modals).
    - `kurir.html`: UI Portal Kurir (form input minimalis, validasi resi, animasi *feedback*).
- **Root Directory**
  - `docker-compose.yml`: Orkestrasi container untuk `web` (Flask backend) dan `mongo` (MongoDB).
  - `.env`: Menyimpan kredensial rahasia (MQTT_PASSWORD, TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID). *(Aman, tidak ter-push ke GitHub).*
  - `.gitignore`: Melindungi `.env` dan dokumen referensi (`aturan.txt`, `PROJECT_PLAN.md`, `prompt.txt`, `reference/`).

### ✅ 2. Fitur yang Sudah Selesai (Phase 1, 2, & 3)
- **Database (MongoDB)**: Manajemen data paket dengan *collection* `packages` (Operasi CRUD penuh).
- **Integrasi Cloud & Notifikasi**:
  - **MQTT Client**: Backend otomatis terkoneksi ke `broker.avisha.id` dan bisa mem-publish perintah `"OPEN"` ke topic `barka/kontrol`.
  - **Telegram Bot**: Mengirim notifikasi otomatis ke *chat id* pemilik saat paket divalidasi.
- **REST API Endpoints**:
  - Manajemen Paket: `/api/tambah_paket`, `/api/edit_paket/<id>`, `/api/hapus_paket/<id>`, `/api/packages`.
  - Interaksi Hardware: `/api/validasi_resi`, `/api/buka_pintu`, `/api/upload_frame` (dummy).
  - Status: `/api/status` (Health check MongoDB & MQTT).
- **Frontend Interaktif**:
  - **Owner Dashboard**: Statistik paket, tabel *auto-refresh*, tombol buka pintu manual, indikator status MQTT, dan form CRUD paket menggunakan **Custom Tailwind Modal** (bukan *alert/prompt* bawaan browser).
  - **Portal Kurir**: Sistem validasi satu *input field* dengan loading state, *shake animation* jika resi salah, dan animasi sukses jika valid.
